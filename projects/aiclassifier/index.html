🤖 case study

Support ticket categorization on autopilot
human in the loop

Problem: team spent 10+ hours/week manually tagging/routing customer emails. Solution: AI classifier + automation pipeline: email → category + priority → task → optional auto-reply.

▶ Live Proof 📄 Docs 📎 SOP 🗺️ Logic ✅ Checks
🧩 before

The friction

  • Inbox triage was slow and inconsistent
  • Priority missed between technicians
  • Repetitive category tagging
Manual work: ~10 hrs/week
✅ after

The fix

  • Email text → category + priority
  • Auto-create task in project tool
  • Auto-reply only for low-priority, confirmed categories
  • Operator review queue for edge cases
Recovered: 12 hrs/week in strategic work

When I talk about this in interviews

1

Name the waste

“Triage by email was costing the team 10+ hours a week on repetitive reads.”

2

Show ownership

“I kept a human-in-the-loop queue for low-confidence or sensitive tickets.”

3

Emphasize fairness

“Classifiers assist routing; they don’t replace the agent with customer contact unless confidence is high.”

4

Tie to result

“The team moved from reactive email triage to structured tasks with accurate priority.”
